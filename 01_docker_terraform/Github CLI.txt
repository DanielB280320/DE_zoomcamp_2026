
github CLI:

  - git init
  - git config --global user.name <DefaultUser>
  - git config --global user.email <DefaultEmail>
  - git config --global init.defaultbranch <branchName>
  - git clone <Repository path> TempRepoName
  - git remote add origin https://github.com/username/repo.git
  - git status
  - gir remote -v
  - git add .
  - git rm -f --cached filename
  - Remove-Item -Recurse -Force foldername/.git
  - git ls-files
  - git commit -am "Add and commit changes"
  - git commit -m "Commit changes"
  - git log
  - git reset --hard <Commit has>
  - git checkout -b <branch/name>
  - git branch -d <branch/name>
  - git branch
  - git branch -M <main or master>
  - git pull origin main 
  - git pull origin main --rebase
  - git pull origin main --allow-unrelated-histories
  - git push origin main 
  - git push -u origin main 

- Note: To avoid conflicts problem when sending info. from our host/local machine to a remote Git Repository, is
        important to review that the data in our local is the same that is hosted in the Remote repository, and also
        we can add new info; For that its recommended first to make a copy of the repository in our local with 'Clone'
        to get the same info that is hosted in the Repo and also add new data to be sended.
